# Что это за проект

Это end-to-end data/ML-проект для управления взысканием задолженности по лицевым счетам. Он берет исходные выгрузки `01..14`, приводит их к единой структуре, строит витрину признаков, сегментирует ЛС, оценивает эффект мер воздействия и готовит материалы для презентации результата.

# Как теперь устроен запуск

Разрозненные этапы объединены в единый пайплайн с одной точкой входа:

```bash
python main.py
```

Эта команда поднимает локальный сайт. В интерфейсе видно весь процесс по шагам, можно запустить полный пайплайн и открыть готовые таблицы, графики и markdown-материалы.

# Что делает пайплайн

1. `prepare_raw`: исходные Excel/CSV превращаются в raw-таблицы.
2. `build_normalized`: raw-данные нормализуются, параллельно собираются DQ-ошибки.
3. `build_mart`: строится витрина признаков без утечки будущей информации.
4. `build_segmentation`: формируются rule-сегменты, ML-кластеры и hybrid business-gates.
5. `build_uplift_prototype`: считается прогнозный uplift по мерам воздействия.
6. `build_presentation_viz`: генерируются графики для слайдов.
7. `build_presentation_notes`: формируются storyline, speaker notes и чеклист.

# Что изменилось

- Командные аргументы убраны из новой точки входа.
- Настройки собраны в `pipeline/settings.py`.
- `pipeline/run_pipeline.py` стал единым оркестратором этапов.
- `pipeline/web_app.py` запускает локальный web-интерфейс.
- `main.py` в корне проекта запускает всю программу.
