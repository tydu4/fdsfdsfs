Структура презентации (12-14 слайдов)
1. Титульный слайд
Название: Персонализированная система рекомендаций мер взыскания (или «Smart Collections: от массовой рассылки к интеллектуальному воздействию»)
Подзаголовок: Решение для хакатона [Название]
Логотипы / Команда

2. Проблема: бизнес-риски и неэффективность ручных правил
Заголовок: Кассовые разрывы и «слепые» воздействия
Текст:

Энергосбыт платит генерации вперед → любая просрочка = кредиты → снижение рентабельности.

Сейчас: жесткие табличные правила (Приложение №1) → не учитывают историю оплат и реакцию на меры.

Ограниченные ресурсы: 200 отключений, 400 судов, 500 выездов в месяц. Часть тратится впустую.

3. Цель и критерии успеха
Заголовок: Что мы сделали и как измеряем результат
Текст:

Цель: Для каждого должника → персонализированный план воздействия на месяц.

Главная метрика: Ожидаемый экономический эффект (руб.) = Вероятность оплаты × Сумма долга - Стоимость меры.

Ограничения: Учтены лимиты, каскады (информирование → ограничение → суд), допустимость (срок/сумма/контакты).

4. Данные и фичи (признаки) — сила не в «сырых» данных, а в производных
Заголовок: Из сырых выписок → в профиль поведения
Текст (буллиты):

Сальдо, начислено, оплачено за 14 месяцев.

История примененных мер (13 типов: автодозвон, суд, отключение…).

Созданные признаки (ключевое!):

Реакция на меру: «оплатил после СМС / после выезда».

Давность просрочки (мес).

Доступные каналы (телефон, email).

Платежная дисциплина (стабильный / хаотичный / ноль оплат).

Результат: каждый должник — вектор из 40+ признаков.

5. Сегментация (кластеризация) — выделяем типовые профили
Заголовок: Не все должники одинаковы: от кошерных до безнадежных
Текст:

Алгоритм: K-means / DBSCAN на основе истории долга и реакции на меры.

Получили 5-6 профилей (пример):

Забывчивые (малый долг, платят после напоминания)
Проблемные с контактом (средний долг, реагируют на звонок)
Злостные без контакта (большой долг, без телефона — только выезд/суд)
Технически сложные (отсутствие движения более года, выезд и ИЛ)
VIP-долг (сумма >30 тыс., высокая вероятность оплаты через суд)
Визуализация: диаграмма распределения по профилям + табло признаков.

6. Модель рекомендаций как задача оптимизации
Заголовок: Предсказываем эффект от каждой меры, выбираем лучшую
Текст:

Для каждого профиля обучается модель uplift / вероятности оплаты по каждой мере.

Расчет Expected Economic Value (EEV) = P(оплаты | мера) × сумма долга – затраты на меру.

Учет:

Допустимость (срок, сумма, телефон/email).

Каскады (нельзя суд без отключения).

Лимиты (глобальная оптимизация greedy / knapsack).

Итог: для каждого ЛС — рекомендованный набор мер на месяц.

7. Пример работы: «было — стало»
Заголовок: От универсального списка к точечному воздействию (кейс)
Таблица / два столбца:

Было (жесткие правила)	Стало (наше решение)
Всем с долгом 500-1500 руб → СМС	Забывчивым → email + автонапоминание
Всем с долгом >1500 руб → обзвон	Проблемным с контактом → автодозвон + 1 оператор
Выезд по формальным критериям	Выезд только тем, кто не реагирует на звонок + >30 тыс.
400 судов в месяц распределены равномерно	400 судов на профили с высокой вероятностью взыскания
Вывод: экономия ресурсов + рост собираемости.

8. Учет ограничений и лимитов (на техническом уровне)
Заголовок: Решение работает в реальных рамках бизнеса
Буллиты:

Жестко закодированы правила из Приложения №1 (мин. срок, сумма, контакты).

Каскады: почта/автодозвон всегда разрешены; ограничение → только после информирования.

Месячные лимиты (например, СМС 2150, выездов 500):

Сначала считаем приоритет по EEV.

Затем greedy allocation с учетом остатка.

Гарантия: рекомендация никогда не нарушит лимит и порядок процедур.

9. Объяснение рекомендации (Explainability) — ключевое для бизнеса
Заголовок: «Почему этому клиенту — суд, а соседу — СМС?»
Текст:

Для каждого должника система выдает карточку решения:

Профиль: «Проблемный без контакта» (кластер 3).

Ключевые признаки: долг 45 тыс. руб., просрочка 9 мес., нет телефона, не реагировал на претензию.

Рекомендация: Выезд → Уведомление об ограничении → Заявление в суд.

Почему не СМС? (нет мобильного номера).

Почему не только суд? (каскад: нужно информирование + ограничение по закону).

Визуализация: SHAP-диаграмма или просто список топ-3 признаков.

10. Интерфейсы (MVP, но честные)
Заголовок: Загрузил данные → получил рекомендации
Что показываем:

Скриншот: форма загрузки файлов (общая информация, ОСВ, оплаты, меры, лимиты).

Кнопка: «Сформировать рекомендации».

Результат: таблица с колонками: ЛС, сумма долга, профиль, меры на месяц, обоснование.

Выгрузка: Excel / CSV для операторов.

Технически: Python + Pandas/Scikit-learn, запуск локально.

11. Результаты и экономический эффект (оценка / A/B-тест в симуляции)
Заголовок: Что мы выигрываем (даже без реального внедрения)
Текст:

На исторических данных проведена симуляция замены правил.

Ожидаемый рост эффективности: +15-25% собираемости при тех же лимитах.

Экономия ресурсов: снижение количества «пустых» выездов и бесполезных судов (пример: 500 выездов начинают приносить на 30% больше оплат).

ROI внедрения: окупаемость 2-3 месяца.

12. Дорожная карта внедрения (взгляд в будущее)
Заголовок: От хакатона до промышленной эксплуатации
Текст:

1 мес: Пилот на одном регионе (с параллельным запуском старых правил).

2-3 мес: Онлайн-дообучение модели, интеграция через API в биллинг.

4 мес: Добавить прогноз денежного потока (когда именно оплатит).

6 мес: Масштабирование на юрлиц.

Текст:

✅ Лимиты и каскады учтены.
